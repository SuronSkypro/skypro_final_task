Перед вами небольшая статья-шпаргалка,  которая включает в себя команды удобные, но не всем знакомые команды Git, которые могут стать вашими постоянными спутниками в разработке.

  1. git init - Инициализирует новый репозиторий git в текущей директории.
  2. git clone <ссылка> - Клонирует указнный репозиторий на локальный компьютер.
  3. git config - Настойка кофигурации среды git.
  4. git remote add <имя> <ссылка> - Добавление удаленного репозитория на локальный компьютер.
  5. git status - Показывет статус локального репозитория, измененные, добавленные а также неотслеживаемые файлы.
  6. git add <Файл> - Добавляет файл для отслеживания изменений.
  7. git reset - Это сложный универсальный инструмент для отмены измененй в проекте.
  8. git commit -m <сообщение> - Добавляет коммит с сохранненными изменениями и прикрепляет его описание.
  9. git push - Используется для отправки фиксаций в локальной ветви на удаленный репозиторий.
  10. git fetch - Используется для скачивания изменений из удаленного или локального репозитория в свой локальный репозиторий.
  11. git merge - Команда выполняет слияние отдельных направлений разработки в единую ветку.
  12. git pull - Забирает изменения из указанного удалённого репозитория, а затем пытается слить их с текущей веткой.
  13. git branch - Главный инструмент для работы с ветвлением, С ее помощью можно добавлять новые ветки, перечислять и переименовывать существующие и удалять их.
  14. git checkout - Команда используется для переключения веток и выгрузки их содержимого в рабочий каталог.
  15. git diff - Команда выводит изменения между вашей текущей рабочей директорией и стейджингом.
  16. git rebase <базовый коммит> - Эта команда для наложения коммитов поверх другого базового коммита.
  17. git stash - Позволяет на время «сдать в архив» (или отложить) изменения, сделанные в рабочей копии, чтобы вы могли применить их позже.
